# Application Modernization Plan

这是维护者视角的应用现代化计划，关注 OpenClaw 如何继续改进架构、插件、控制 UI、运行时和部署方式。

普通读者可以把它理解成：项目不是一次性写完，而是不断把旧结构搬到更清晰、更安全、更可扩展的新结构上。

## 重点方向

- 把能力收进插件和 manifest 边界。
- 让 Control UI、CLI、Gateway 之间的协议更稳定。
- 让密钥、权限、沙箱和节点模型更清楚。
- 让旧入口保留兼容，但新教程引导到新入口。

## 新手怎么读

如果你只是安装使用，可以跳过。
如果你想读源码或做二次开发，把它当作“未来装修图纸”：它解释为什么有些旧结构会慢慢搬家。
