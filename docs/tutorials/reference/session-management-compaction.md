# Session Compaction

会话太长时，OpenClaw 需要压缩上下文，保留重点，丢掉不需要留意的细节。

目标不是忘记一切，而是让模型还能在上下文窗口里继续工作。

继续阅读：[上下文压缩](/tutorials/concepts/compaction)。

## 为什么需要压缩

模型一次只能看有限上下文。会话越来越长时，如果不整理，后面的消息可能挤不进去。

压缩会把“有用事实、当前目标、已经做过的事”留下，把重复寒暄、无关输出和过长日志丢掉或概括。

## 新手提醒

压缩不是备份。需要留意文件和长期知识应该放进工作区、Wiki 或记忆系统，不要只依赖聊天上下文。
